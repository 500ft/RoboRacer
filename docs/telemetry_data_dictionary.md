# Telemetry Data Dictionary

## Purpose

This document defines the columns in `runs/first_lap/telemetry.csv` so the first-lap evidence run can be interpreted without reading the logging script. The telemetry is generated by `experiments/run_scripted_lap.py` from F1TENTH Gym observations, pure-pursuit commands, and derived nearest-waypoint metrics.

## Dataset Context

| Field | Value |
| --- | --- |
| Dataset | `runs/first_lap/telemetry.csv` |
| Metadata | `runs/first_lap/metadata.json` |
| Track | `examples/example_map` |
| Waypoints | `examples/example_waypoints.csv` |
| Controller | pure pursuit |
| Logged integrators | `rk4`, `euler` |
| Simulation timestep | 0.01 s in the first-lap evidence run |

## Column Definitions

| Column | Units | Sign convention / values | Source | Used for |
| --- | --- | --- | --- | --- |
| `run_id` | unitless | `first_lap_<integrator>` | experiment script | grouping rows by run |
| `integrator` | unitless | `rk4` or `euler` | experiment script | comparing numerical integration behavior |
| `step` | count | starts at 1 after the first environment step | experiment script | ordering samples |
| `time_s` | s | nonnegative elapsed simulation time | `step * reward`; reward equals environment timestep | time-aligned plots and metrics |
| `x_m` | m | global map-frame x position | Gym observation `poses_x` | trajectory and position error |
| `y_m` | m | global map-frame y position | Gym observation `poses_y` | trajectory and position error |
| `theta_rad` | rad | global yaw angle from map x-axis | Gym observation `poses_theta` | heading and kinematic replay |
| `speed_mps` | m/s | positive forward speed in logged runs | Gym observation `linear_vels_x` | speed tracking and replay state |
| `steer_rad` | rad | actual steering state after constraints/dynamics | `env.sim.agents[0].state[2]` | comparing commanded vs applied steering |
| `command_speed_mps` | m/s | pure-pursuit speed command | planner output | replay input / controller trace |
| `command_steer_rad` | rad | positive command follows Gym steering convention | planner output | replay input / controller trace |
| `yaw_rate_radps` | rad/s | positive counterclockwise yaw rate | Gym observation `ang_vels_z` | dynamic-model diagnostics |
| `accel_x_mps2` | m/s^2 | positive forward acceleration | finite difference of `speed_mps` over timestep | acceleration diagnostics and replay input candidate |
| `accel_y_mps2` | m/s^2 | computed as `speed_mps * yaw_rate_radps` | experiment script | lateral-acceleration diagnostic |
| `nearest_waypoint_index` | index | zero-based waypoint segment index | nearest-waypoint projection in experiment script | progress and CTE traceability |
| `progress_m` | m | distance along waypoint/raceline progress coordinate | interpolated from waypoint column 0 | progress-aligned metrics |
| `cte_m` | m | signed cross-track error from waypoint segment; sign follows segment cross product | experiment script | tracking-error analysis |
| `abs_cte_m` | m | nonnegative absolute cross-track error | `abs(cte_m)` | summary metrics and plots |
| `lap_time_s` | s | Gym lap timer value | Gym observation `lap_times` | lap completion diagnostics |
| `lap_count` | count | Gym lap count value | Gym observation `lap_counts` | lap completion diagnostics |
| `collision` | boolean int | `0` no collision, `1` collision | Gym observation `collisions` | termination and validation |
| `termination_reason` | enum string | `completed_lap`, `collision`, `max_steps`, `error`, or blank before final propagation | experiment script | filtering successful vs failed runs |

## Notes and Open Items

- `command_steer_rad` is the controller-requested steering angle; `steer_rad` is the simulator's steering state after steering-rate limits and dynamics.
- `accel_x_mps2` is a finite-difference estimate from logged speed, not a direct accelerometer measurement.
- `accel_y_mps2` is a derived diagnostic using \(a_y \approx v r\), not a direct simulator sensor channel.
- The exact positive steering/yaw convention for model replay must be checked against `docs/vehicle_model.md` before comparing signs.
- Any new telemetry columns added by later experiments must be appended here before the data is used as evidence.
